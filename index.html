Written Question

[1] Describe how your dashboard answers the questions presented. You don't have to address every question directly, but should at a high level address the main questions. (10 points)

Answer:
The first question ask me about the top 10 games for each year. At first, I set a criterion that best game is the game which has the highest global sales. Since the global sale is a real number, the "barplot" is one of the best way to compare different values. In order to show the results for different years, the dynamic "barplot" is adopted. After filling in the year of interest, the user can check the corresponding top 10 games with their global sales in that year. Instead of setting a x-coordinate, I inserted the value of the global sales next to the bar.

The second question ask me about the popular genre for each region. At first, I set a standard that the popular genre is the genre which has the highest regional sales. I use "Map", together with "tooltip", to show the "Popular Genre per Region". In addition, I use the circle-area to represent the value of the regional sales and use circle-color to represent the corresponding genre. What's more, I inserted a textbox which shows clearly the region name and the popular genre. If the user want to know more details, he can even move his mouse to the corresponding circle and more information will be extracted. The result showes that the most popular genre in North America, Europe and other regions is Action. The most popular genre in Japan is Role-Playing.

I use "boxplot", together with "tooltip", to present the "Best Publisher per Genre". As a player, I know that there are several standards for the evaluatin of a company (such as "Total Number of Game Selling", "The sum of gobal sales for all the games", "The average sales per game", "The global sales for the most popular game"). Therefore, I provide the choice for the user to see the results based on different standard. There are too many publishers for each genre, therefore I only keep the top "10" publisher based on the standard. In order to see whether a publisher is a little better or much better than its components. I calculate the statistical "Min", "1/4 First quartile", "Median" "3/4 Third quartile" and "Max". These information can help the boss to understand the relative score for the publisher.

[2] List 3 reasons why D3 was helpful and improved your visualization (6 points)
Answer:
a) The dynamic plot and interactive tooltip will provide more information about the data than a static plot (such as Originlab and Excel)! They can show more than one images and the user can extract more information from the data.
b) The D3 has great flexibility/freedom. Instead of using a template, we can easily combine different elements (such as barplot, circle, pie, etc) and create the best image for visuallization.
c) There are a lot of fantastic D3 projects online. I can easily find good samples and improve my own visualizaton process.

[3] List 3 reasons why D3 would not be the best tool for creating a visualization (6 points)
Answer:
[a] Difficult to learn. The coding part for d3 is too tedious. The new user has to spend a lot of time on learning the functions of D3 (even Javascript!). In addition, the browser does not always provide useful information when debuging.
[b] Privacy. It is difficult for me to hide my data source in the D3. The user can get information about my data-link or cvs file.
[c] Platform. Some old browsers may not be able to handle d3. Since Javascript engine is needed, the environment for the d3 in Mobile Phone, tablet, switch/PS5/XBOX, machines, FPGA, etc. should be considered carefully.


[4]Evaluate the accessibility of your dashboard based on the readings in the “Before you begin” section. What kinds of users might find this dashboard accessible and who might have more difficulty? What additional actions might you take to make this dashboard more accessible to all audiences? Your response should refer to at least one of the readings and be about 1 paragraph.
Answer:
My dashboard has great accesibility. As introduced by the second article that we should "Label data points directly."), I showed the value of data directly next to the bar which makes it easy to read. In the Graph-2 (Map plot), I marked both the location and the name for each region. In the Graph-3, I showed the best publisher directly on the figure and the user has freedom to get more statistical information about the top 10 publishers. The colorblindness will find the dashboard easy to read, because I use both the length of bar and the area of circle to represent the value of the data. The old people with trembling hands will find the dashboard hard to read, because different data points are too close to each other for the user to hover on. As suggested by the article "A Comprehensive Guide etc.", "Don’t require a user to hover over a graph to see the data output. This is often difficult or impossible to do when using assistive technology. Instead, show the output at all times." The tooltip in Graph 3 is not convenient because of the huge amount of data points. I should avoid mixing a lot of data points together but be focus on the most important information.
